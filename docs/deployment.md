# LinkNote Deployment Notes

This document consolidates the current deployment shape for LinkNote.

## Local Development

Backend:

```bash
uvicorn api_server:app --reload --port 8000
```

Convenience script:

```bash
sh run_api.sh
```

Desktop:

```bash
cd desktop
npm install
npm run tauri dev
```

The desktop Tauri shell starts the FastAPI backend from the local LinkNote path and uses the same API server as the browser UI.

## Render Deployment

The repo contains `render.yaml` for Blueprint deployment.

Render start command:

```bash
uvicorn api_server:app --host 0.0.0.0 --port $PORT
```

Render environment shape:

- `PYTHON_VERSION=3.11.9`
- `EMBED_PROVIDER=openai`
- `DATA_DIR=/var/data`
- `CHROMA_PATH=/var/data/chroma_db`
- `AUTH_SECRET` generated by Render
- `OPENAI_API_KEY` entered manually in the Render dashboard
- `GOOGLE_CLIENT_ID` entered manually in the Render dashboard

Render disk:

- `mountPath: /var/data`
- `sizeGB: 1`
- Required for persistent `data/` and ChromaDB storage.

## Local Data Versus Deployed Data

Local data and deployed data are separate.

Local default paths:

- `./data`
- `./data/uploads`
- `./chroma_db` or `CHROMA_PATH` when set

Render paths:

- `/var/data`
- `/var/data/uploads`
- `/var/data/chroma_db`

Do not assume local ChromaDB state can be copied directly into production. Embedding provider differences matter. In particular, Ollama embeddings and OpenAI embeddings should not be mixed in the same Chroma collection.

## Required Secrets

Do not commit secrets.

Required or optional runtime secrets:

- `OPENAI_API_KEY`: required for OpenAI answer generation, concept extraction, and OpenAI embedding modes.
- `GOOGLE_CLIENT_ID`: required for Google login.
- `AUTH_SECRET`: required for stable token signing in deployed environments.

The current provider code is designed so the app can start even when `OPENAI_API_KEY` is absent, but OpenAI-dependent actions should fail gracefully with clear messages.

## Git Hygiene

The current `.gitignore` excludes:

- `.env`
- `venv/`
- `.venv312/`
- `chroma_db/`
- `data/`
- `__pycache__/`
- `*.pyc`
- `.DS_Store`
- desktop build artifacts

This means local user data, uploaded PDFs, generated concept JSON, auth storage, and vector indexes should not be pushed to GitHub.
