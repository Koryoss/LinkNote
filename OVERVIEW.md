# LinkNote 개요

## 1. 서비스 목적
강의 PDF를 올리면 과목·단원·개념으로 정리해 보여주고, 질문하면 자료를 근거로 답하는 개인 학습 앱.
핵심은 **서로 다른 과목의 개념을 연결**해 공부하게 돕는 것.

## 2. 기존 해결책과의 차별점
- 일반 PDF 챗봇: 한 문서 안에서만 답함.
- LinkNote: **과목을 넘나드는 개념 연결**(예: 병태생리 "신부전" ↔ 인체구조 "신장")을 임베딩 그래프로 찾아줌.
- 미술관 같은 **"지식 갤러리"** UI(과목 액자·책장·개념지도).

## 3. 진행 단계
- ✅ 완료: 갤러리 UI(액자·책장·개념지도), 시간표, PDF 업로드, 개념 추출, 과목 간 연결 그래프,
  계정(이메일+구글)·보안, 데스크톱 .app(Tauri), 질문 2종(단일/연결) 개선.
- 🔄 지금: 웹 배포(Render 무료) 진행 중 + 로컬 임베딩을 OpenAI로 전환 중.
- ⏭ 다음: 웹 배포 완료(유료 디스크로 데이터 영구저장) → 질문 정확도 추가 튜닝.

## 4. 기술 스택 (버전별 차이)
- 공통: FastAPI(백엔드) + ChromaDB(자료 벡터 저장) + 화면은 `gallery.html` 한 장.
- 답변 생성: **OpenAI**(데스크톱·웹 둘 다).
- 임베딩(검색용 벡터): `EMBED_PROVIDER` 로 선택.
  - 데스크톱: 기본 Ollama(로컬·무료, 한국어 약함) → **OpenAI로 전환 진행 중**.
  - 웹: OpenAI(정확).
- 데스크톱: Tauri로 감싼 .app, 앱이 백엔드 자동 실행. 구글 로그인 ✕(웹뷰 차단) → 이메일 로그인.
- 웹: Render 호스팅, 어디서나 접속, 구글 로그인 ○.

## 5. 그 외 핵심
- 비용: OpenAI(답변·임베딩) 소액 유료, Render 디스크 월 ~$7. Ollama·구글로그인 무료.
- 데이터 분리: 로그인 사용자별로 자기 자료만 보임(토큰으로 서버가 강제).
- 한계: 로컬 검색이 한국어에 약함(질문 시 과목 지정으로 우회) → OpenAI 임베딩이 근본 해결.
- 역할 분담: 엔진(rag/검색)=Codex, 화면·문서=Claude (`TEAM_ROLES.md`).
- 데이터 흐름: PDF → 텍스트 추출 → 조각(청크) → 벡터 저장 → 질문 시 비슷한 조각 찾아 OpenAI가 답.
